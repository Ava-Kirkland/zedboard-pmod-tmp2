# Pmod TMP2 Temperature Sensor — ZedBoard

Read temperature from a Digilent Pmod TMP2 (ADT7420) over I2C, using the Zynq PS I2C0 peripheral routed via EMIO to JB Pmod connector pins. Temperature is printed to UART in both Celsius and Fahrenheit.

**Tools:** Vivado 2025.2.1 · Vitis 2025.2  
**Board:** ZedBoard (Zynq-7000)  
**Sensor:** Pmod TMP2 (ADT7420, I2C)  
**Protocol:** I2C (IIC) via EMIO, polled mode

## Key Tips

> These are the lessons that cost the most time. Read before starting.

**1. Make the IIC_0 interface external as a bus — not the individual pins.**  
Right-click the `IIC_0` interface port on the PS block and select **Make External**. Vivado auto-generates the IOBUFs and collapses SDA/SCL into two bidirectional ports. If you instead make the 6 `_I/_O/_T` scalar ports external individually, you must manually write IOBUF instantiations in the wrapper — a significant extra step with room for error.

**2. Rename the external IIC port — remove the trailing `_0`.**  
After making IIC_0 external, Vivado names it `IIC_0_0`. Click the port and rename it to `IIC_0` in the External Interface Properties tab. This affects the port names that appear in synthesis and your XDC must match exactly.

**3. Pull-ups are mandatory. Not optional.**  
I2C is open-drain. Without pull-ups, SDA and SCL never return high and the bus is dead. Set `PULLTYPE PULLUP` in the XDC for both pins, or the sensor will not respond.

**4. Every new `.xsa` requires a new Vitis workspace.**  
Updating an existing platform component with a new `.xsa` in the same workspace has not produced a working result. Create a fresh `ws/` directory for each hardware export. See [bugs_and_fixes.md](docs/bugs_and_fixes.md).

**5. The Pmod TMP2 has no power LED.**  
Use a multimeter to confirm 3.3V between VCC and GND on the JB connector before attaching the Pmod. Do not hot-plug.

**6. Direct plug-in to JB (W8/W10) is unresolved.**  
Using W8 (SCL) and W10 (SDA) — which would allow the Pmod to plug directly into JB without jumper cables — did not work during testing. Root cause is unknown. Use jumper cables to W12 (SCL) and W11 (SDA) until this is investigated. See [bugs_and_fixes.md](docs/bugs_and_fixes.md) Bug #3.

**7. Negative temperature path is untested.**  
The ADT7420 uses two's complement for sub-zero readings. The current driver has only been validated on positive temperatures. The conversion math may produce incorrect results below 0°C. See [docs/project_walkthrough.md](docs/project_walkthrough.md).
